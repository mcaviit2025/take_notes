Section 1.1

Information Assurance Fundamentals

Information Assurance (IA) is the practice of managing risks related to the use, processing, storage, and transmission of information and information systems. IA ensures that information and systems are protected from both internal and external threats across five core properties — Confidentiality, Integrity, Availability, Authentication, and Non-repudiation. The technologies covered in this section — cryptography, DNS, firewalls, virtualisation, and RFID — are the foundational tools of information assurance.
Topic 1.1.1

Basic Cryptography

Cryptography is the science of transforming readable information (plaintext) into an unreadable form (ciphertext) to protect it from unauthorised access, and of transforming it back to readable form for authorised recipients. It is the mathematical foundation of all information security.
📌 Last Night Tip
Three essential terms: Plaintext (original readable message), Ciphertext (encrypted output), Key (the secret used in encryption/decryption). Know the difference between symmetric (one key) and asymmetric (two keys: public + private) — it is the most frequently examined distinction.

Core Cryptographic Terminology

TermDefinitionExample
Plaintext (P)The original, readable message before encryption"Transfer ₹50,000 to account 1234"
Ciphertext (C)The scrambled, unreadable output after encryption"X7#kQz!&9mBp@Lv..."
Encryption (E)The process of converting plaintext to ciphertext using an algorithm and keyC = E(K, P)
Decryption (D)The reverse process — converting ciphertext back to plaintextP = D(K, C)
Key (K)The secret value that controls the encryption/decryption processA 256-bit binary string: 1011010...
Algorithm / CipherThe mathematical procedure (recipe) used to perform encryptionAES, RSA, DES, Blowfish
CryptanalysisThe art of breaking ciphertext without the key — "cracking" encryptionBrute force, differential cryptanalysis
Hash FunctionA one-way transformation that maps data to a fixed-length digest; not reversibleSHA-256 of "Hello" = 185f8db3...

Goals of Cryptography — CIA + AN

🔒

Confidentiality

Only authorised parties can read the information. Encryption ensures ciphertext is meaningless to anyone without the key.

✅

Integrity

Data has not been altered in transit. Digital signatures and hash functions detect any modification — even a single bit change.

🆔

Authentication

Verifies the identity of the sender. Digital certificates and message authentication codes (MACs) confirm "this message really came from Alice."

🚫

Non-Repudiation

The sender cannot later deny sending the message. Digital signatures cryptographically bind the sender to the message — proof is mathematically verifiable.

📶

Availability

Cryptographic systems themselves must be available when needed. Key management and certificate revocation must not create single points of failure.

Types of Cryptography

Fig 1.1 — Cryptography Classification Tree

Hash Functions — One-Way Cryptography

A cryptographic hash function takes an input of any length and produces a fixed-length output (digest). It is one-way — you cannot reverse the hash to get the original input. Even a tiny change in input produces a completely different hash (the avalanche effect). Hash functions are used for password storage, file integrity verification, and digital signatures.

// Hash Function — Avalanche Effect Demonstration
Input 1: "Hello"
SHA-256: 185f8db32921bd46d35490...a38e45c0

Input 2: "hello" ← changed ONE character (capital H → lowercase h)
SHA-256: 2cf24dba5fb0a30e26e83b...8a9af62

// Completely different output — that is the avalanche effect
✔ Hash functions are deterministic: same input → always same output
✖ MD5 is cryptographically broken — NEVER use for security purposes

Topic 1.1.2

Symmetric Encryption

Symmetric Encryption uses the same single key for both encrypting and decrypting data. The sender and receiver must share this secret key through a secure channel before communicating. It is fast and efficient — ideal for encrypting large volumes of data — but the key distribution problem (how do you securely share the key?) is its principal weakness.

How Symmetric Encryption Works

Fig 1.2 — Symmetric Encryption: One Shared Key

Types of Symmetric Ciphers

Block Ciphers vs Stream Ciphers

Block Cipher

  • Encrypts data in fixed-size chunks (blocks) — typically 64-bit or 128-bit blocks
  • Each block is processed independently (or chained together)
  • Slower but more secure for general-purpose encryption
  • Modes of operation: ECB, CBC, CTR, GCM (each with different security properties)
  • Examples: AES, DES, 3DES, Blowfish, Twofish, IDEA

Stream Cipher

  • Encrypts one bit or byte at a time — generates a keystream that is XOR'd with plaintext
  • Extremely fast — ideal for real-time communication (voice, video)
  • Synchronisation between sender and receiver is critical
  • Vulnerable if the same keystream is reused — must never reuse nonce/IV
  • Examples: RC4, ChaCha20, Salsa20, A5/1 (GSM)

Major Symmetric Encryption Algorithms

AES

Advanced Encryption Standard

The global standard since 2001. Chosen by NIST after an open competition. Replaced DES. Used in HTTPS, Wi-Fi (WPA2/WPA3), banking, and virtually every modern secure system.

128 / 192 / 256-bit keys 128-bit block size Unbroken FIPS 197
DES

Data Encryption Standard

The original US government standard (1977). Uses only a 56-bit key — considered insecure today. A 56-bit key can be brute-forced in under 24 hours with modern hardware. Replaced by 3DES and then AES.

56-bit key — INSECURE 64-bit block size Deprecated
3DES

Triple DES

Applies DES three times with different keys to increase security. Offered a migration path from DES. Slow compared to AES. NIST deprecated 3DES in 2023 — no longer recommended for new systems but still found in legacy banking infrastructure.

112 / 168-bit effective key 64-bit block Deprecated 2023
Blowfish

Blowfish Cipher

Designed by Bruce Schneier (1993) as a free, fast alternative to DES. Variable key length — very flexible. Still used in some legacy systems. Superseded by its successor Twofish and AES. Known for its slow key setup — advantageous for password hashing (bcrypt uses Blowfish).

32–448-bit variable key 64-bit block bcrypt (passwords)
RC4

Rivest Cipher 4 (Stream)

Widely used in SSL and WEP Wi-Fi encryption. Extremely fast and simple. However, discovered to have significant cryptographic weaknesses — WEP is trivially crackable using RC4 vulnerabilities. RC4 is now prohibited in TLS 1.2+ and should never be used for new implementations.

Stream cipher — BROKEN Prohibited in TLS 1.3 WEP cracked in minutes
ChaCha20

ChaCha20-Poly1305 (Stream)

Modern stream cipher designed by Daniel Bernstein. Extremely fast in software (no hardware acceleration needed — ideal for mobile devices). Used in TLS 1.3, WireGuard VPN, and Android's default HTTPS cipher. Combined with Poly1305 MAC for authenticated encryption.

256-bit key TLS 1.3 standard WireGuard VPN

Key Distribution Problem — The Critical Weakness

The fundamental challenge of symmetric encryption: how do Alice and Bob securely share the secret key before they can encrypt anything? If they transmit it over an insecure channel, an eavesdropper (Eve) captures it and can decrypt all future messages. This problem is solved by asymmetric encryption (covered in Topic 1.1.3).

⚠️ AES Key Size Matters
AES-128 offers 2128 possible keys — computationally infeasible to brute-force with current technology. AES-256 is recommended for data requiring long-term protection (classified government documents, medical records). For most commercial applications, AES-128 or AES-256 in GCM mode is the recommended choice.

Topic 1.1.3

Public Key (Asymmetric) Encryption

Public Key Cryptography (Asymmetric Encryption) uses a mathematically linked key pair — a Public Key (shared openly with everyone) and a Private Key (kept secret by its owner). Data encrypted with the public key can only be decrypted with the matching private key, and vice versa. This elegantly solves the key distribution problem of symmetric encryption.
📌 Most Examined Concept
Public key = anyone can use it to ENCRYPT or VERIFY. Private key = only the owner uses it to DECRYPT or SIGN. Lock and key analogy: the public key is a padlock you hand to everyone; the private key is the only key that opens it. Know RSA, the formula C = Me mod n, and how digital signatures work.

The Key Pair — How It Works

Fig 1.3 — Asymmetric Encryption: Two Operations (Encrypt vs Sign)

RSA — The Most Widely Used Asymmetric Algorithm

RSA (Rivest–Shamir–Adleman, 1977) is the foundational public-key algorithm. Its security relies on the mathematical difficulty of factoring the product of two large prime numbers. Given N = p × q (where p and q are large primes), finding p and q from N is computationally infeasible for large enough N.

// RSA — Simplified Key Generation and Encryption

Step 1 — Choose primes: p = 61, q = 53
Step 2 — Compute n: n = p × q = 61 × 53 = 3233 (public modulus)
Step 3 — φ(n): φ(n) = (61-1)(53-1) = 3120
Step 4 — Choose e: e = 17 (coprime with φ(n); public exponent)
Step 5 — Find d: d = 2753 (d × e ≡ 1 mod φ(n); private exponent)

Public Key: (e=17, n=3233) — share with everyone
Private Key: (d=2753, n=3233) — keep secret always

Encrypt M=65: C = M^e mod n = 65^17 mod 3233 = 2790
Decrypt C=2790: M = C^d mod n = 2790^2753 mod 3233 = 65 ✔

Hybrid Encryption — Best of Both Worlds (How HTTPS Works)

Modern systems use a hybrid approach — asymmetric encryption to securely exchange a symmetric key, then symmetric encryption for all actual data transfer. This gives the security of asymmetric with the speed of symmetric.

1 Server Certificate Server sends its RSA/ECC public key in a digital certificate signed by a trusted CA
2 Key Exchange Client generates a random AES session key; encrypts it with server's public key; sends it
3 Session Key Server decrypts using private key; both now share the same AES session key
4 Data Transfer All HTTPS data encrypted with fast AES-256-GCM using the shared session key
5 Session End Session key discarded; new session = new key. Forward secrecy maintained

Topic 1.1.4

The Domain Name System (DNS)

The Domain Name System (DNS) is the internet's distributed, hierarchical naming system that translates human-readable domain names (like google.com) into machine-readable IP addresses (like 142.250.195.78). DNS is often called the "phone book of the internet" — without it, every website would have to be accessed by its numerical IP address. DNS is also a critical security component — and a frequent attack target.

DNSSEC — DNS Security Extensions
DNSSEC adds cryptographic signatures to DNS records. A DNS resolver can verify that a response came from the authoritative server and has not been tampered with. DNSSEC uses public/private key pairs — the zone signing key (ZSK) signs individual records, and the key signing key (KSK) signs the ZSK. India's .in domain supports DNSSEC, and CERT-IN recommends all government portals implement DNSSEC.

Topic 1.1.5

Firewalls

A Firewall is a network security device (hardware, software, or cloud-based) that monitors and controls incoming and outgoing network traffic based on a set of predefined security rules. It establishes a barrier between a trusted internal network and untrusted external networks (the internet), permitting legitimate traffic and blocking malicious or unauthorised traffic.

Types of Firewalls

Packet Filtering

Stateless Inspection

Examines each packet in isolation — checks source/destination IP, source/destination port, and protocol (TCP/UDP). Fast but cannot understand context — does not know if a packet is part of an existing legitimate connection.

Layer 3–4 (Network/Transport) Fast but limited No state tracking
Stateful Inspection

Connection-Aware Firewall

Tracks the state of active network connections. Knows whether a packet is part of an established session (SYN-ACK flow tracking). Allows responses to legitimate outgoing requests. Much smarter and more secure than packet filtering.

Layer 3–4 + state table Most common type iptables, pfSense, ASA
Application / Proxy FW

Layer 7 Inspection

Operates at the application layer — understands specific protocols (HTTP, FTP, DNS, SMTP). Acts as a proxy — terminates client connection, inspects content, then establishes new connection to server. Can block malicious payloads within allowed protocols.

Layer 7 (Application) Deep Packet Inspection Slower but thorough
Next-Gen Firewall

NGFW — All-in-One

Combines stateful inspection + DPI + integrated IPS + application awareness + SSL/TLS inspection + user identity awareness + threat intelligence feeds. Identifies applications by behaviour, not just port numbers. The modern enterprise standard.

All layers Palo Alto · Fortinet · Cisco App-ID · User-ID
WAF

Web Application Firewall

Specifically protects web applications from Layer 7 attacks: SQL injection, cross-site scripting (XSS), CSRF, path traversal, and OWASP Top 10 vulnerabilities. Deployed in front of web servers. Can be hardware, software, or cloud-based (CDN-integrated).

HTTP/HTTPS only Cloudflare · AWS WAF · ModSecurity OWASP Rules
Cloud Firewall / FWaaS

Firewall as a Service

Cloud-native firewalls that protect cloud infrastructure (VPCs, subnets) and distributed workforces (zero-trust architecture). Scale elastically with traffic. Examples include AWS Security Groups, Azure NSG, and cloud-based SASE solutions (Zscaler, Prisma).

AWS SG · Azure NSG Zscaler · Prisma SASE Elastic scaling

Firewall Rule Sets — How Rules Work

Firewall rules are processed top-to-bottom. The first matching rule wins — subsequent rules are not evaluated. Every firewall ends with an implicit or explicit deny all rule at the bottom (default deny / whitelist approach). This is the safest configuration.

Topic 1.1.6

Virtualization

Virtualization is the technology that creates a software-based (virtual) representation of physical resources — servers, storage, networks, and operating systems. A single physical machine can run multiple isolated virtual machines (VMs) simultaneously, each with its own OS, applications, and network stack. Virtualization is the foundation of cloud computing and plays a critical role in information security through isolation, sandboxing, and snapshot-based forensics.

Virtualization Architecture — The Hypervisor

The Hypervisor (Virtual Machine Monitor — VMM) is the software layer that manages and allocates physical resources (CPU, RAM, storage) to virtual machines. There are two types:

Type 1 — Bare-Metal Hypervisor

  • Runs directly on the hardware — no host OS underneath
  • More efficient and secure — smaller attack surface
  • Used in enterprise data centres and cloud platforms
  • VMs are fully isolated from each other
  • Examples: VMware ESXi, Microsoft Hyper-V, Xen, KVM
  • Used by: AWS (Xen/Nitro), Google Cloud (KVM), Microsoft Azure (Hyper-V)

Type 2 — Hosted Hypervisor

  • Runs on top of a conventional host operating system (Windows/Linux/macOS)
  • Host OS manages hardware; hypervisor runs as an application
  • Less efficient — additional abstraction layer introduces overhead
  • Used on desktop machines for development and testing
  • Examples: VMware Workstation, VirtualBox, Parallels Desktop
  • Used by: Developers, security analysts, students (MCA lab environments)
Fig 1.6 — Virtualisation Stack: Type 1 vs Type 2 Hypervisor

Security Benefits of Virtualization

1
Isolation
Each VM is fully isolated from other VMs. A malware infection in VM1 cannot spread to VM2 without exploiting the hypervisor itself (VM escape — very rare)
Malware lab in VM, clean office in host
2
Snapshots
Take a point-in-time snapshot before risky operations. If something goes wrong, revert instantly. Critical for malware analysis, forensics, and patch testing
Forensic VM snapshot before analysis
3
Sandboxing
Run untrusted code in an isolated VM. Observe its behaviour — file changes, network calls, registry modifications — without risking the real system
Any.run, Cuckoo Sandbox malware analysis
4
Reduced Attack Surface
Server consolidation reduces the number of physical machines exposed to the network. Unused VMs can be powered off — attack surface shrinks instantly
10 services on 1 hypervisor vs 10 physical servers
5
Disaster Recovery
VM images can be replicated to remote data centres in real time. If a data centre is destroyed or ransomwared, VMs can be started at the DR site within minutes
VMware Site Recovery, Azure Site Recovery
⚠️ Virtualisation Security Risk — VM Escape
A VM escape attack exploits a vulnerability in the hypervisor to break out of the VM and access the host OS or other VMs. This is the most dangerous virtualisation attack. Notable examples: VENOM (2015) — a flaw in QEMU's virtual floppy drive that allowed VM escape from any Linux/Xen guest. Always keep hypervisor software patched.

Topic 1.1.7

Radio-Frequency Identification (RFID)

RFID (Radio-Frequency Identification) is a technology that uses electromagnetic fields to automatically identify and track tags attached to objects. An RFID system consists of a tag (a microchip + antenna), a reader (emits radio waves and receives tag response), and a backend database (stores information associated with the tag ID). RFID is ubiquitous in supply chains, access control, contactless payment, and inventory management.

RFID System Components

Fig 1.7 — RFID System Architecture

RFID Tag Types by Power Source

📡

Passive RFID Tags

No internal battery. Power is harvested from the reader's electromagnetic field to activate the chip and transmit a response. Very cheap (₹5–₹50 each), thin, lightweight, and have an almost unlimited lifespan. Cannot initiate communication — only respond when in range of a reader.

Range: 10cm – 7m · Freq: 125 kHz, 13.56 MHz, 860–960 MHz (UHF)
🔋

Active RFID Tags

Have their own battery — can continuously broadcast their ID signal. Much longer range and more memory capacity. Used for tracking large assets (shipping containers, vehicles, livestock, hospital equipment). Battery life typically 3–7 years; more expensive (₹500–₹5,000 each).

Range: 30m – 100m+ · Freq: 433 MHz, 915 MHz, 2.45 GHz
⚡

Semi-Passive (BAP) Tags

Have a battery for powering the chip and memory, but use the reader's field for communication (like passive tags). Longer range than passive, longer battery life than active. Used in temperature-sensitive supply chains (pharmaceuticals, food), cold chain monitoring.

Range: 10m – 50m · Freq: UHF 860–960 MHz

🇮🇳 India — FASTag (RFID on National Highways)
FASTag is India's RFID-based electronic toll collection system mandated by NHAI (National Highways Authority of India). A passive RFID sticker (operating at 860–960 MHz UHF) is affixed to the vehicle windshield. As the vehicle passes through a toll plaza, an overhead reader reads the tag and deducts the toll from a linked bank account/wallet automatically. Over 97% of toll transactions on national highways are now cashless via FASTag. Security risks include: tag tampering, relay attacks, and fraudulent FASTag stickers. NHAI uses cryptographic authentication of tag reads to mitigate fraud.

Section 1.2

Microsoft Windows Security Principles

Microsoft Windows is the world's most widely deployed desktop operating system — and therefore its most targeted. Windows implements a sophisticated security architecture built around the concepts of tokens (representing identity and privileges), messaging (inter-process communication), program execution controls (UAC, DEP, ASLR), and a built-in firewall. Understanding these mechanisms is essential for both securing Windows environments and understanding how attacks exploit them.
Topic 1.2.1

Windows Tokens

A Windows Access Token is a kernel object created by the Windows Security Subsystem (LSASS) when a user successfully logs on. It acts as a digital identity badge that travels with every process and thread the user runs. The token contains the user's Security Identifier (SID), group memberships, privileges, and the security context for the session. Every access control decision (can this process open this file? run this service?) is made by comparing the token's contents against the object's security descriptor.

Token Types and Impersonation

🏅

Primary Token

Assigned to a process when it is created. Represents the default security context for all threads in that process. Created at logon and passed to child processes. Every running application (notepad.exe, chrome.exe) has a primary token determining what it can and cannot access.

whoami /all ← shows current token contents
GetCurrentProcessToken() ← Win32 API call
🎭

Impersonation Token

Assigned to a thread to temporarily act with a different user's security context — without starting a new process. Critical for client–server applications: an IIS web server thread can impersonate the client's identity to access resources the client has permission to access, while the server process runs as a higher-privilege service account.

ImpersonateLoggedOnUser() ← impersonate client
RevertToSelf() ← return to own identity
🔑

Restricted Token

A modified copy of a primary token with reduced privileges — specific privileges and group SIDs are removed or marked as "deny only." Used by Windows for sandboxing: Google Chrome's renderer processes run with a restricted token to limit damage if the renderer is exploited. Created using CreateRestrictedToken().

CreateRestrictedToken() ← create sandbox token
Used by: Chrome, Edge, Adobe sandbox
🛡️

Filtered (Split) Token — UAC

When a standard user account that is also a local administrator logs on, Windows creates two tokens: a filtered standard token (Medium integrity, no admin groups) for everyday use, and a full administrator token (High integrity) that is only activated when the user explicitly consents to UAC elevation. This is the core of User Account Control.

Medium Token → normal processes
High Token → activated via UAC consent prompt

Topic 1.2.2

Windows Messaging

Windows Messaging is the inter-process communication (IPC) mechanism at the heart of the Windows GUI. Every window (button, text box, dialog box, application window) communicates through messages — integer-identified packets sent between processes, threads, and system components. The Windows message subsystem enables applications to communicate, respond to user input, and interact with the OS — but it also introduces significant attack surfaces.

The Windows Message System — Architecture

Every GUI thread in Windows has a message queue. When a user clicks a button, moves the mouse, or presses a key, the Windows kernel generates messages and places them into the relevant thread's message queue. The application's message loop continuously retrieves messages and dispatches them to the appropriate Window Procedure (WndProc) for handling.

// Simplified Windows Message Loop — Every GUI Application Has This

MSG msg;
while (GetMessage(&msg, NULL, 0, 0)) ← retrieves next message from queue
{
    TranslateMessage(&msg); ← translates keyboard input to WM_CHAR
    DispatchMessage(&msg); ← sends message to WndProc handler
}

// Common Windows Messages:
WM_PAINT 0x000F ← window needs repainting
WM_KEYDOWN 0x0100 ← a keyboard key was pressed
WM_LBUTTONDOWN 0x0201 ← left mouse button clicked
WM_CLOSE 0x0010 ← user clicked the X button
WM_COMMAND 0x0111 ← menu item or button was activated

Message Sending Functions

FunctionBehaviourSecurity Implication
SendMessage()Sends a message to a window and waits for the target window to process it before returning (synchronous)Can be used for cross-process attacks — send crafted message to another process's window to trigger unintended behaviour
PostMessage()Places a message in the target window's message queue and returns immediately (asynchronous)Can inject messages into another process's queue — used by keyloggers and UI automation attacks
BroadcastSystemMessage()Sends a message to all top-level windows in the system simultaneouslyUsed by malware to notify all running processes of a system-wide event (e.g., fake shutdown)

Windows Messaging Security Attacks

⌨️

Keylogging via Hooks

SetWindowsHookEx(WH_KEYBOARD) installs a system-wide hook that intercepts every keyboard message before it reaches its target application. Malware uses this to log all keystrokes — passwords, messages, financial data — across all applications. Requires matching process architecture (32/64-bit).

SetWindowsHookEx(WH_KEYBOARD_LL, hookProc, hMod, 0);
← intercepts ALL keystrokes system-wide
🖱️

Shatter Attacks

Cross-privilege messaging attack discovered in 2002. A low-privilege process sends a crafted WM_TIMER or WM_COPYDATA message to a high-privilege window (running as SYSTEM). The high-privilege window's WndProc executes malicious code provided in the message — escalating the attacker's privileges to SYSTEM. Mitigated by UIPI (UI Privilege Isolation) in Vista+.

🪟

UI Privilege Isolation (UIPI)

Windows Vista+ defence against Shatter attacks. UIPI prevents lower-integrity processes from sending messages to higher-integrity windows. A Medium integrity browser cannot send messages to a High integrity UAC prompt. This prevents credential dialog spoofing and cross-privilege message injection. ChangeWindowMessageFilterEx() can selectively relax restrictions.

📋

Clipboard Hijacking

Malware registers as a clipboard viewer using AddClipboardFormatListener(). Every time the user copies anything to the clipboard (password from a manager, bank account number, cryptocurrency address), the malware receives a WM_CLIPBOARDUPDATE message and reads — or silently modifies — the clipboard contents before the user pastes.

Topic 1.2.3

Windows Program Execution

Understanding how Windows loads and executes programs is fundamental to both offensive security (how malware plants itself) and defensive security (what controls exist to prevent unauthorised code from running). Windows implements multiple overlapping security mechanisms around program execution — from User Account Control (UAC) to memory protection (ASLR, DEP) to code signing.

Windows Program Execution Flow

1 User Initiates Double-click .EXE / command typed in CMD / scheduled task fires
2 Shell / Explorer Explorer.exe calls CreateProcess(); application manifest read; UAC check performed
3 Token Assignment Process assigned primary token (Medium or High integrity depending on UAC outcome)
4 Loader (NTDLL) NT loader maps EXE + DLLs into virtual address space; resolves imports; applies ASLR
5 DEP / CFG Data Execution Prevention marks data pages non-executable; Control Flow Guard initialised
6 Code Executes Main thread starts at entry point; application runs under its assigned token and integrity level

Key Windows Execution Security Mechanisms

🛡️

User Account Control (UAC)

Introduced in Windows Vista. When a standard user or administrator tries to run a program requiring elevated privileges, UAC displays a consent prompt (blue = signed publisher; orange = unknown publisher). This prevents programs from silently elevating to administrator without user knowledge. Applications must declare elevation requirements in their application manifest.

Manifest: <requestedExecutionLevel level="requireAdministrator">
UAC bypass techniques: DLL hijacking, token impersonation, fodhelper.exe
🎲

ASLR — Address Space Layout Randomisation

Randomises the base addresses of the executable, stack, heap, and loaded DLLs in virtual memory every time a process starts. Makes it extremely difficult for attackers to predict where code or data will reside — invalidating hardcoded memory addresses in exploit shellcode. Must be enabled in both the OS and the application's linker flags (/DYNAMICBASE).

Without ASLR: kernel32.dll ALWAYS at 0x7C800000
With ASLR: kernel32.dll at 0x77A4F000 or 0x754E0000 (random)
🚫

DEP — Data Execution Prevention

Marks memory regions that contain data (stack, heap) as non-executable. If an attacker overflows a buffer and places shellcode in a data region, the CPU will raise an exception when execution reaches it — preventing classic buffer overflow exploitation. Implemented in hardware (Intel XD bit, AMD NX bit) and enforced by Windows kernel.

DEP bypass: Return-Oriented Programming (ROP)
— chains existing executable code gadgets instead of injecting new code
🔐

Code Signing and SmartScreen

Windows requires that software drivers are signed by Microsoft. SmartScreen Filter checks downloaded executables against Microsoft's reputation database. Unsigned or unknown programs receive a prominent warning. Windows Defender Application Control (WDAC) / AppLocker can restrict execution to only digitally signed, approved applications — the strongest execution control available.

AppLocker rule: Allow only code-signed by "ACME Corp" certificate
WDAC policy: Block all unsigned PowerShell scripts

DLL (Dynamic Link Library) Security

Windows applications rely heavily on DLLs for shared code. This creates attack opportunities through DLL hijacking — placing a malicious DLL in a location that Windows searches before the legitimate DLL location.

DLL Search Order — Windows Default
Windows searches for DLLs in this order: (1) DLLs already loaded in memory → (2) System DLLs (KnownDLLs registry) → (3) Application directory → (4) System32 → (5) Windows directory → (6) Current working directory → (7) PATH environment variable directories.

DLL Hijacking attack: If an application loads version.dll from its directory before System32, an attacker with write access to the application directory can place a malicious version.dll there — it loads first, running the attacker's code with the application's privileges.

Topic 1.2.4

The Windows Firewall

Windows Firewall with Advanced Security (WFAS) — also known as Windows Defender Firewall — is the built-in, stateful, host-based firewall integrated into all versions of Windows since XP SP2 (2004). Unlike perimeter network firewalls, it protects the individual Windows machine — filtering traffic at the host level, regardless of what network the computer is connected to. It is a mandatory component of Windows security architecture and cannot be disabled without a replacement firewall.

Windows Firewall Profiles

Windows Firewall applies different rule sets depending on the detected network type. Windows automatically selects the appropriate profile based on the network location category:

🏢 Domain Profile

Active Directory Network

Applied when the computer is connected to a domain network (corporate). Typically least restrictive — domain controllers, file servers, and management tools need to communicate freely. Rules managed by Group Policy from the domain administrator.

Corporate / Office network GPO managed
🏠 Private Profile

Trusted Home Network

Applied when the user designates a network as private (home or small office). Allows local network discovery, file sharing, and printing — assuming other devices on the network are trusted. More permissive than Public.

Home / SOHO network Network discovery ON
☕ Public Profile

Untrusted Network

Applied when connected to a public network (café, airport, hotel Wi-Fi). Most restrictive — blocks all inbound connections by default. Network discovery disabled. Assumes all other devices on the network are hostile. The default profile for new unknown networks.

Public Wi-Fi / untrusted Inbound: Block All by default

Windows Firewall Rule Types

Rule TypeDirectionWhat It ControlsExample
Inbound Rules→ Into the machineControls what connections can be made TO this computerAllow Remote Desktop (TCP 3389) from 10.0.0.0/8 only
Outbound Rules← Out of the machineControls what connections this computer can make OUTBlock all outbound on port 445 (SMB) to external IPs — prevents WannaCry lateral spread
Connection Security RulesBothEnforces IPsec authentication/encryption for connections matching the ruleRequire IPsec for all connections from domain computers to SQL server

Windows Firewall Rule Criteria

Each firewall rule can match on multiple criteria. The rule only fires when all specified criteria are matched:

  • Program: A specific executable (e.g., allow only C:\Program Files\Teams\Teams.exe to use port 443)
  • Port: Source or destination TCP/UDP port number or range
  • Protocol: TCP, UDP, ICMPv4, ICMPv6, or any
  • IP Addresses: Source and/or destination IP address or subnet
  • Interface Type: Wired, wireless, or VPN connections only
  • Service: A specific Windows Service (e.g., only the Windows Update service)
  • User / Group: Only apply this rule when the connection is made by a specific domain user or group (requires IPsec authentication)

Managing Windows Firewall — Key Methods

🖥️

Windows Defender Firewall GUI

Accessible via Control Panel → System and Security → Windows Defender Firewall. Simple on/off and basic program exceptions. For advanced rules, use "Windows Defender Firewall with Advanced Security" (wf.msc) — full inbound/outbound rule management with all criteria.

wf.msc ← opens Advanced Security console
⚡

PowerShell / netsh

Command-line management for scripting and automation. netsh advfirewall is the legacy command; New-NetFirewallRule PowerShell cmdlet is the modern approach. Essential for GPO scripts and automated deployment of firewall rules across enterprise environments.

New-NetFirewallRule -DisplayName "Block Telnet" `
  -Direction Inbound -Protocol TCP `
  -LocalPort 23 -Action Block
🏛️

Group Policy (GPO)

Centrally manage Windows Firewall rules across all domain-joined computers. Computer Configuration → Windows Settings → Security Settings → Windows Defender Firewall. Policies pushed from domain controller override local firewall settings — essential for enterprise standardisation and incident response (block a port organisation-wide within minutes).

GPO path: Computer Config → Windows Settings
→ Security Settings → Windows Defender Firewall
📊

Firewall Logging

Windows Firewall can log all allowed and/or dropped connections to %SystemRoot%\System32\LogFiles\Firewall\pfirewall.log. Logs include timestamp, action (ALLOW/DROP), protocol, source IP, destination IP, ports, and packet size. Critical for forensic investigation and incident detection. Log file size and location configurable via WFAS.

Log location: C:\Windows\System32\LogFiles\Firewall\pfirewall.log
Event Viewer: Security log, event IDs 5156 (allow), 5157 (block)

Windows Firewall in Action — WannaCry Containment (India, 2017)

India · 2017 · Ransomware Containment · Host-Based Firewall
The Attack Vector

WannaCry exploited the EternalBlue vulnerability in Windows SMB (Server Message Block) protocol, which operates on TCP port 445. The worm spread automatically to any Windows machine on the same network with port 445 open and unpatched — no user interaction required.

How Windows Firewall Could Have Contained It
  • Organisations with Windows Firewall blocking inbound TCP 445 from external networks were protected — WannaCry could not reach them
  • Even on internal networks, a host-based Windows Firewall rule blocking SMB between workstations (allowing only to file servers) would have prevented lateral movement
  • Indian organisations that quickly deployed an outbound block on TCP 445 via GPO stopped the worm from reaching new victims within their network
The GPO Command That Saved Networks
# Block SMB inbound from any source except trusted file servers
New-NetFirewallRule -DisplayName "Block WannaCry SMB" -Direction Inbound -Protocol TCP -LocalPort 445 -Action Block -Profile Any
Key Lesson Host-based firewalls are the last line of defence when perimeter firewalls fail or are absent. A single GPO-deployed Windows Firewall rule blocking unnecessary SMB exposure could have prevented billions of rupees in damages during the WannaCry outbreak. Every Windows endpoint should have the firewall enabled and configured — not just left at defaults.
Unit 1 — Final Takeaway

Cyber security essentials span two complementary domains: the mathematical (cryptography — AES, RSA, hashing, key exchange, PKI) and the architectural (firewalls, DNS, virtualisation, RFID, Windows security internals). Neither is sufficient alone. An unencrypted system behind a perfect firewall is still vulnerable to insider theft; a perfectly encrypted system with a poorly configured DNS server can be hijacked before a connection is ever established. Information security requires a layered, defence-in-depth approach — applying controls at every layer of the technology stack, from the cryptographic algorithms in your HTTPS handshake to the firewall rules on your Windows workstation.
